=== OBC インタフェース（CON1）
CON1 は OBC との接続インタフェースです。 +

搭載コネクタ：SEAM-40-02.0-S-10-1-A-K-TR（Samtec）

=== BPD 電源供給用インタフェース（CON2）
CON2 は Versal の Battery Power Domain (BPD) に対し、電源を供給することが可能なインタフェースです。 +
BPD は Real Time Clock (RTC) と Battery Backup RAM (BBRAM) を包括しています。 +
RTC と BBRAM は、通常は OBC 内で生成される電圧 (VCCAUX_PMC_SMON) により動作していますが、OBC の主電源が遮断された場合、CON2 から供給される電圧で維持、または保持されます。

=== Versal / IGLOO2 デバックインタフェース（CON6）
CON6 は Versal 及び IGLOO2 のデバックインタフェースです。 +
FTDI IC が搭載されており、以下のデバック方法の選択が可能です。

- Versal FPGA JTAG
- Versal APU/RPU Console
- IGLOO2 Safety Processor Telemetry/Telecommand

また CON6 には、双方向高速マルチプレクサを経由して、Versal の 高速シリアル通信信号である GTYP が接続されており、Versal SmartLynq+ HSDP でのデバックも可能です。 

==== Versal FPGA JTAG
Versal の FPGA JTAG インタフェースです。 +
FTDI IC の Channel. A に接続されており、JP13~JP18 に挿入されているジャンパーソケットを『FTDI』シルク側へ切り替えることで利用が可能です。

==== Versal APU/RPU Console
Versal の APU/RPU のコンソールです。 +
FTDI IC の Channel. B および C に、Versal の UART0 および UART1 にマッピングされたシリアル通信信号が接続されています。 +
ただし MIO のテストで信号を確保できなくなる場合に、Versal PL HDIO に UART0, UART1 をマッピング（EMIO）することを考え、JP19~JP21 および JP22~JP24 で MIO と EMIO の切り替えが可能にしています。

==== IGLOO2 Safety Processor Telemetry/Telecommand
IGLOO2 の UART を利用し、OBC からの Telemetry 信号によるモニタ、または OBC への Telecommand 信号によるモード変更が可能です。 +
JP25~JP29 を FTDI IC の Channel. D へ切り替えることで利用が可能です。

==== Versal SmartLynq+ HSDP
Versal の HSDP (High-Speed Debug Port) を使用し、AMD社製 SmartLynq+ モジュールを介して高速デバックおよびトレースが可能です。 +
JP34 を切り替えることで利用が可能です。

=== Versal Reserve インタフェース（CON7）
CON7 は Versal の Bank501 に属する MIO 信号が繋がっています。 +
CON7 に繋がる MIO は、SD カードコネクタとの間にレベルシフタを設ける場合、その通信方向を制御する目的で使用されます。 +
I/Fボードでは双方向通信が可能なレベルシフタを採用しており、本機能は不要となるため Reserve としています。

=== microSD カードインタフェース（CON8）
CON8 は microSD カードを挿入できるインタフェースです。 +
起動するためのソフトウェアを、OBC 上の Flash メモリや eMMC ではなく、microSD カードに書き込んで使用することを想定しています。

=== USB2.0 インタフェース（CON9）
CON9 は OBC を USB2.0 のホストとして使用可能なインタフェースです。 +
VBUS は、I/Fボードの主電源から生成された +5V が Load Switch を経由して出力されます。最大負荷は約 1A です。

=== Gigabit Ethernet インタフェース（CON10）
CON10 は RJ45 型ジャックの Gigabit Ethernet インタフェースです。 +
Versal の APU へ Linux OS をポーティングすることで、OBC へのリモートログインが可能となります。 +
接続のリンクが確立した場合、CON10 に搭載されている緑色の LED が点灯します。 +
また通信中は、同じく CON10 に搭載されている黄色の LED が点滅します。 +
CON10 を使用する場合、Ethernet として使用している Versal MIO の電源ドメインが 1.8V 以上となるように JP38~JP41 を選択してください。

=== FMC モジュール接続用インタフェース（CON11）
CON11 は Versal PL または IGLOO2 FPGA 信号の内、高速シリアル通信信号を評価する目的で、市販の FMC モジュールを接続することが可能なインタフェースです。

=== Versal MIO 拡張インタフェース（CON12）
CON12 は、OBC 上の Versal Bank 500~502 に属する MIO の内、CON1 に接続され、CON8 (microSD) , CON9 (USB2.0) , CON11 (Ethernet) で使用されていない MIO にアクセスできるインタフェースです。 +
CON12 の Pin 1~8 に接続されている MIO は、JPxx により CON12 以外の回路へ接続が可能となるため注意が必要です。 +
CON12 に繋がる MIO の 電源ドメインである VDD_B502 は、JP38~JP41 で選択が可能です。 +
VDD_B502 には、Versal の主電源が落ちた場合、確実に Bank 500~502 への電源も落とすため、電源ラインには Load Switch が設けられています。 +
CON12 の Pin 13~14 にあたる PMC_MIO12, PMC_MIO37 は、それぞれ Versal Bank 500 と Bank 501 に接続される MIO であり、電源ドメインが 1.8V で固定されていることから、他の MIO と電源ドメインを揃える目的でレベルシフタ (IC27) を介しています。 +
また Pin 13~14 の通信方向に従って JP47~JP48 を切り替える必要があることに注意してください。

=== Versal POR 制御スイッチ（SW2）
SW2 は Versal の External POR (Power On Reset) を制御可能なタクトスイッチです。 +
SW2 を押すことで、Versal の全てのロジックとレジスタがリセットされ、PMC Start-up Flowchart に従って再起動します。

SW3 は IGLOO2 の POR (Power On Reset) を制御可能なタクトスイッチです。 +
SW3 を押すことで IGLOO2 の FPGA Fabric はリセットされ、Power-Up to Functional Time Sequence に従って再起動します。

=== I/F ボード主電源切り替えスイッチ（SW4）
SW4 は I/F ボードの主電源の ON/OFF の切り替えが可能なスライドスイッチです。 +
『ON』のシルクの方向へトグルを移動することで主電源が回路へ供給され、『OFF』のシルクの方向へ移動することで供給を遮断することが可能です。

LED4 は FMC モジュールへの電源供給状態を示す黄緑色のインジケータです。 +

LED5 は主電源の供給状態を示す黄緑色のインジケータです。 +

SW4 のトグルが『OFF』のシルク側へ移動した場合、消灯します。 +

LED6~LED9 はユーザーが任意で使用が可能な黄緑色の LED です。 +
詳細は CON17 を参照してください。

=== Versal GTYP セレクタ（JP34）
JP34 は Versal の高速シリアル通信信号である GTYP の接続先の切り替えが可能なインタフェースです。 +
CON6 を利用して SmartLynq+ モジュールによる Versal の高速デバックおよびトレースを行いたい場合は、ジャンパーソケットを挿入してください。 +
上記を行わず、CON11 に接続した FMC モジュールで評価を行いたい場合は、ジャンパーソケットを挿入しない状態にしてください。
